1.定时器产生中断的过程：系统时钟 → 分频器（Prescaler） → 降低后的时钟 → 计数器（CNT）
计数器 CNT 每来一个时钟脉冲就 +1，CNT 达到 ARR （自动重装载寄存器）时，CNT清零并重新开始计数

2.RCC全称`Reset and Clock Control`复位与时钟控制，外设运行必须先开RCC时钟，更准确得说：把这个外设接上工作节拍，让它能响应寄存器配置并开始运行
注意点（1）为什么这么设计，不能默认全开？ 

> 答：为了省电，为了降低干扰，提高系统可靠性

注意点（2）看到STM32外设初始化主要有三个步骤：

> 第一步：这个外设挂在哪个总线上？（AHB,APB2,APB1）；
> 第二步：它的时钟开了吗？；
> 第三步：除了总线时钟，它还有没有专用时钟？如ADC 的 ADCCLK、RTC 的时钟源、USB 的 48MHz 时钟

3.串口常用电平标准：
**TTL电平**：+3.3V或+5V表示1，0V表示0
**RS232电平**：-3~-15V表示1，+3~+15V表示0
**RS485电平**：两线压差+2~+6V表示1，-2~-6V表示0(差分信号

4.STM32主函数必须叫main，因为单片机刚上电后先执行Reset_Handler函数；启动代码默认会调用名为 main 的函数，所以主函数通常必须叫 main，不能直接叫 main1，除非你自己改启动代码或在 main 里调用 main1。

5.注意STM32F103常见启动模式

| BOOT0 | BOOT1 |    启动位置     |          用途          |
| :---: | :---: | :-------------: | :--------------------: |
|   0   | 任意  |    主 Flash     | 正常运行自己烧录的程序 |
|   1   |   0   | 系统 Bootloader |   串口 ISP 下载程序    |
|   1   |   1   |      SRAM       | 很少用，调试/特殊启动  |

需要理解的是：
- BOOT0和BOOT1是跳线帽连接的，具体三个针脚通常是3.3V--BOOTx--0V，所以跳线帽可以使其置0或1；
- 主Flash就是程序可以运行main函数。ISP（In-System Programming），先进入厂家预置好的 **Bootloader 程序**，由它接收电脑发来的程序，然后写入 Flash，要运行该程序需要回到第一种状态

6.注意，硬件角度有两个DR（RDR和TDR），但是软件映射是只有一个DR的

| 外设  | 是否全双工 | CPU 可见 DR  | 硬件功能上的数据缓冲  | 移位寄存器理解                |
| ----- | ---------- | ------------ | --------------------- | ----------------------------- |
| USART | 是         | 1 个 DR 地址 | TDR + RDR 两个        | TX 移位寄存器 + RX 移位寄存器 |
| SPI   | 是         | 1 个 DR 地址 | TX 缓冲 + RX 缓冲两个 | 1 个移位寄存器边移出边移入    |
| I2C   | 否，半双工 | 1 个 DR      | 1 个 DR 复用          | 1 个移位寄存器收发复用        |

7.在线编程ICP用于更新程序存储器的全部内容，它通过JTAG、SWD协议或者系统加载程序（系统存储区的Bootloader）下载程序，
在程序中编程（IAP）也就是自己写的Bootloader程序，存储于0x08000000，此时0x08004000才是APP 程序
